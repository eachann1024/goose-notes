# 功能边界

- 开始任务前，先判断目标属于速记小窗、常规笔记本或两者共享，再定位实现。
- 速记小窗是独立的轻量草稿便签，入口为 `quicknote.html`，使用 `__GOOSE_LITE__` 和 `quicknote.css` 隔离功能与样式。
- 常规笔记本是完整工作区，入口为 `index.html`，包含笔记管理、侧栏、标签页、AI 和文档级能力。
- 用户明确提到“小窗”“速记”“便签”时，默认只修改速记小窗，不得改变常规笔记本行为。
- 用户明确提到“笔记本”“主窗”“工作区”时，默认只修改常规笔记本，不得改变速记小窗行为。
- 只有用户明确要求两端一致，或功能天然属于共享编辑器能力时，才修改共享层。
- 修改共享组件前，必须检查两个入口的调用路径，并用显式模式分支或入口专属样式保持产品边界。
- 交互归属按使用场景判断，不按代码复用关系判断。窄窗空间优化、贴边工具栏、草稿槽位和置顶行为属于小窗；笔记管理、页面导航和 AI 工作区属于常规笔记本。

# 验证

- 修改代码后以 `bun run build` 通过为验收标准。
- 默认不要用浏览器、截图或页面交互做验证。
- 仅当用户明确要求视觉验收、交互复现或 UI 对比时，才使用浏览器。
